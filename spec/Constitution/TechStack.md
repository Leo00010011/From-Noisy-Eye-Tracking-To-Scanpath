# Tech Stack

## Runtime Environment

- **Python**: 3.x (check `.python-version` or `conda env` for exact version)
- **Hardware**: CUDA GPU required for MixerModel training; CPU fallback is supported but slow
- **OS**: Developed on Windows; some config paths are Windows-style and must be adjusted on Linux/macOS
- **Key local paths that must be configured before running**:
  - `model.image_encoder.repo_path` — absolute path to the local DINOv3 repository clone
  - `model.image_encoder.weights` — filename of the DINOv3 weights `.pth` file (placed alongside the repo or in the working directory)
  - Dataset root: defaults to `data/Coco FreeView/` relative to the project root; override with the `LOCAL_SCRATCH` env var (HPC scratch usage)
- The HDF5 file (`data/Coco FreeView/dataset.hdf5`) must be generated by the preprocessing pipeline before any training. It is not version-controlled.

## Architecture Overview

Two model families share the same training infrastructure and config system.

### PathModel (`src/model/path_model.py`)
Gaze-only encoder–decoder transformer. Takes the noisy gaze trajectory (`src`) and autoregressively predicts clean fixations (`tgt`). No image features. Used as an ablation baseline and for fast iteration. Input encoders: `linear` or `shared_gaussian`.

### MixerModel (`src/model/mixer_model.py`)
Full image-conditioned architecture. Extends PathModel with:
- A frozen DINOv3 image encoder (`DinoV3Wrapper`) that extracts per-patch features
- A `FeatureEnhancer` (cross-attention between gaze encoding and image patches) **or** an `eye_decoder` (cross-attention in the opposite direction — gaze queries attend to image patches)
- A `DoubleInputDecoder` (or deformable variant) that attends to both image features and the encoded gaze trajectory during fixation prediction
- A `denoise_head` (MLP) that runs on the encoder output and predicts denoised coordinates

The model partitions its submodules into `denoise_modules` and `fixation_modules`. The active phase (`Denoise` / `Fixation` / `Combined`) toggles `requires_grad` on these groups and routes the forward pass accordingly.

**Current default config** (`configs/model/mixer_model.yaml`):
- `n_encoder=4`, `n_eye_decoder=4`, `n_decoder=4`, `model_dim=512`, `n_heads=8`
- `input_encoder="shared_gaussian"` — separate Gaussian Fourier encoders for 2D coords, timestamp, and duration; weights are shared between encoder and decoder paths
- `use_deformable_eye_decoder=True`, `use_deformable_fixation_decoder=True` — deformable attention in both cross-attention stages
- `image_encoder.name="dinov3_vits16"`, frozen

| Class | File | Role |
|---|---|---|
| `PipelineBuilder` | `src/training/pipeline_builder.py` | Single entry point that reads the Hydra config and constructs every component: dataset, splits, dataloaders, model, optimizer, scheduler, loss, and training utilities. The source of truth for which transforms and model variants are actually wired up. |
| `MixerModel` | `src/model/mixer_model.py` | Full image-conditioned model. Maintains `denoise_modules` / `fixation_modules` lists; `set_phase()` freezes the inactive group. `load_encoder()` cherry-picks encoder weights from a checkpoint by module name. |
| `PathModel` | `src/model/path_model.py` | Gaze-only baseline. Simpler: single encoder stack → decoder stack → head. `set_phase()` is a no-op. |
| `DinoV3Wrapper` | `src/model/dino_wrapper.py` | Wraps a locally cloned DINOv3 model. Returns per-patch tokens including a CLS prefix token. Loaded from a local `.pth` file, not from HuggingFace at runtime. |
| `FreeViewInMemory` | `src/data/datasets.py` | **Primary dataset class.** Loads all scanpath data from the HDF5 file into RAM at startup. Applies the configured transform pipeline per `__getitem__`. Paired with `seq2seq_padded_collate_fn` for variable-length sequence batching. |
| `DeduplicatedMemoryDataset` | `src/data/datasets.py` | Image dataset that avoids loading the same stimulus image multiple times. Used when `use_img_dataset=True`. |
| `CoupledDataloader` | `src/data/datasets.py` | Synchronises iteration over the gaze dataset (`FreeViewInMemory`) and the image dataset (`DeduplicatedMemoryDataset`), yielding matched (gaze, image) batch pairs. |
| `CocoFreeView` | `src/data/parsers.py` | Parses the raw CocoFreeView JSON/image files. Provides split helpers (`get_disjoint_splits`, `get_stimuly_disjoint_splits`) and subject/stimulus enumeration used by `PipelineBuilder.make_splits()`. |
| `ExtractRandomPeriod` | `src/data/transforms.py` | Crops a random temporal window from the full gaze trajectory and aligns it with the fixation subsequence that fits entirely within that window. First transform in the pipeline; determines `x` (noisy gaze) and `y` (clean fixations). |
| `AddRandomCenterCorrelatedRadialNoise` | `src/data/transforms.py` | Physics-informed WebGazer noise: correlated radial error with a drifting center. Primary noise model. |
| `AddIsotropicGaussianNoise` | `src/data/transforms.py` | Simpler isotropic Gaussian noise; used as an alternative or in combination. |
| `DiscretizationNoise` | `src/data/transforms.py` | Quantises gaze coordinates to integer pixel positions to simulate WebGazer discretisation. |
| `Normalize` | `src/data/transforms.py` | Divides spatial coords by image size and timestamps by period duration. Has an `inverse()` method used during evaluation to recover pixel-space coordinates. |
| `LogNormalizeDuration` / `QuantileNormalizeDuration` | `src/data/transforms.py` | Normalises fixation durations. `LogNormalizeDuration` is active; `QuantileNormalizeDuration` is an alternative requiring a pre-fitted `quantile_transformer.pkl`. |
| `SaveCleanX` | `src/data/transforms.py` | Snapshot of the gaze trajectory before noise is added; stored as `clean_x` in the sample dict. Required for the `Denoise` training phase. Must appear before noise transforms in the list. |
| `EntireRegLossFunction` | `src/model/loss_functions.py` | Combined regression + classification loss treating coords and duration as a single regression target. |
| `SeparatedRegLossFunction` | `src/model/loss_functions.py` | Separate loss terms for (x,y) coordinates, duration, and end-of-sequence classification. Default for the fixation phase. |
| `CombinedLossFunction` | `src/model/loss_functions.py` | Wraps a denoise loss and a fixation loss; `denoise_weight` controls the balance. Used in the Combined phase. |

| `GaussianFourierPosEncoder` | `src/model/pos_encoders.py` | Random Fourier feature encoding for continuous coordinates. Used in `shared_gaussian` and `fourier` input encoder modes. |
| `RopePositionEmbedding` | `src/model/rope_positional_embeddings.py` | RoPE for the cross-attention between gaze trajectory and image patches. Active only when `use_rope=True`. |
| `eval_metrics.py` | `src/eval/` | `eval_reg` (Euclidean coord error + duration MAE on masked positions), `eval_denoise` (MSE on denoised coords), `accuracy` / `precision` / `recall` for end-of-sequence head. |
| `eval_utils.py` | `src/eval/` | Offline evaluation utilities: running inference on a split and aggregating metrics. |
| `vis_scanpath.py` | `src/eval/` | Plotting predicted vs. ground-truth scanpaths overlaid on the stimulus image. |

## Data Flow

A single training sample travels through these stages:

1. **HDF5 read** (`FreeViewInMemory`): loads raw gaze array `x` shape `[3, T_full]` (rows: x_pixel, y_pixel, timestamp_ms) and fixation array `y` shape `[3, N_full]` (rows: x_pixel, y_pixel, duration_ms), plus a `fixation_mask` array.

2. **Transform pipeline** (applied in `__getitem__`):
   - `ExtractRandomPeriod` → `x`: `[3, T]`, `y`: `[3, N]` (random window)
   - `SaveCleanX` (if present) → snapshots `x` as `clean_x` before noise
   - Noise transforms (`AddRandomCenterCorrelatedRadialNoise`, `DiscretizationNoise`, etc.) → `x` now contains simulated WebGazer noise
   - `Normalize(key='x', mode='coords')` → `x[:2]` in `[0,1]`; `Normalize(key='x', mode='time')` → `x[2]` in `[0,1]`
   - `Normalize(key='y', mode='coords')` → `y[:2]` in `[0,1]`; duration normalised separately

3. **Collation** (`seq2seq_padded_collate_fn`): pads variable-length sequences to the batch maximum. Output tensors: `src` `[B, T, 3]`, `tgt` `[B, N, 3]`, padding masks.

4. **Model forward** (`MixerModel.forward`):
   - `encode(src, image_src, src_mask)`: projects gaze tokens → `model_dim`, adds positional encoding, runs through `path_encoder` layers, then cross-attends with image patches via `feature_enhancer` or `eye_decoder`
   - `decode_fixation(tgt, ...)`: projects clean fixation tokens (teacher-forced during training), prepends start token, runs through `decoder` layers attending to both image features and encoded gaze
   - `decode_denoise()` (Denoise/Combined phases only): MLP head on encoder output → denoised `[B, T, 2]`

5. **Loss and metrics**: `CombinedLossFunction` applies denoise loss (L1 on `clean_x[:,:,:2]`) and fixation loss (`SeparatedRegLossFunction` on coord + duration + end-of-sequence). `eval_reg` and `eval_denoise` from `eval_metrics.py` compute validation metrics.

**Shape conventions**: `[B, T, 3]` for gaze trajectories, `[B, N, 3]` for fixation sequences, `[B, H*W, D]` for image patch tokens (with one CLS prefix token). Coordinates are always `(x, y)` normalised to `[0,1]`.

## Configuration System

Hydra composes configs from groups. The root is `configs/main.yaml`:

```
defaults:
  - scheduler: warmup          # LR schedule
  - data: default              # dataset and transform params
  - model: mixer_model         # model architecture
  - loss: separated_loss       # loss function
  - head_type: multi_mlp       # output head
  - exp: whole_model_pretraining  # high-level experiment overrides
```

**Config groups and what they control**:
- `configs/model/`: `mixer_model.yaml` (full model), `path_model.yaml` (ablation)
- `configs/scheduler/`: `warmup.yaml` (warmup-stable-decay), `one_cycle.yaml`, `multistep_lr.yaml`
- `configs/loss/`: `entire_loss.yaml`, `separated_loss.yaml`, `focal_loss.yaml`
- `configs/head_type/`: `multi_mlp.yaml` (separate coord/dur/end heads), `mlp.yaml`, `linear.yaml`, `argmax_regressor.yaml`, `heatmap.yaml`
- `configs/data/split_strategy/`: `stimuly_disjoint.yaml` (stimuli-disjoint split), `disjoint.yaml` (subject+stimuli disjoint), `random.yaml`
- `configs/exp/`: experiment-level overrides that set `training.Phases`, epoch counts, etc.

**Most commonly overridden keys**:
- `model.n_encoder`, `model.n_decoder`, `model.model_dim`, `model.n_heads`
- `training.Phases` — controls which phases run and in what order
- `model.image_encoder.enabled` — set to `False` to switch to PathModel-equivalent behaviour
- `model.pretrained_encoder_path` — path to a pre-trained denoise checkpoint, or `"auto_best_denoise"` to auto-select the best run from `outputs/`
- `model.input_encoder` — `"shared_gaussian"` (current default), `"linear"`, `"fourier"`, etc.

Hydra writes each run's resolved config to `outputs/<date>/<time>/.hydra/config.yaml`. That file is the source of truth for reproducibility — do not edit it post-run.

## Training Phases

The phase system allows staged training of MixerModel submodules. Phases are listed in `training.Phases` and executed in order.

| Phase | Modules trained | Loss | Decisive metric |
|---|---|---|---|
| `Denoise` | `denoise_modules` (encoder + denoise head) | L1 on denoised coords vs. `clean_x` | `denoise_error_val` |
| `Fixation` | `fixation_modules` (decoder + prediction heads) | `SeparatedRegLossFunction` (coord + dur + end) | `reg_error_val` |
| `Combined` | All modules | `CombinedLossFunction` with `denoise_weight` | `reg_error_val` |

A typical full training run uses `["Fixation", "Combined"]` (default in `main.yaml`). Denoise pretraining (`["Denoise"]`) is run separately and the checkpoint is loaded via `model.pretrained_encoder_path`.

**Scheduled sampling** (`ScheduledSampling`): during fixation/combined phases, gradually replaces teacher-forced decoder inputs with the model's own predictions. Controlled by `scheduled_sampling.*` config keys.

## Evaluation Protocol

Validation runs every `training.val_interval` epochs. Metrics computed on the validation split:

- `reg_error_val` — mean Euclidean distance in normalised `[0,1]` space between predicted and ground-truth fixation coordinates (masked to valid positions)
- `dur_error_val` — mean absolute duration error in normalised space
- `denoise_error_val` — mean Euclidean distance between denoised output and `clean_x` (only during Denoise/Combined phases)
- `cls_accuracy_val`, `cls_precision_val`, `cls_recall_val` — end-of-sequence head performance

**Before computing pixel-space metrics**, invert the `Normalize` transforms using their `.inverse()` methods. The model outputs and ground truth are in `[0,1]` normalised space; multiply by `(image_W, image_H)` to recover pixels.

Offline batch evaluation uses `src/notebooks/eval_batch.py` and `src/notebooks/generate_review_notebooks.py`.

## Important Nuances / Gotchas

- **DINOv3 paths must be set**: `model.image_encoder.repo_path` and `model.image_encoder.weights` are hardcoded to a local Windows path in `configs/model/mixer_model.yaml`. Override these before running on any other machine.
- **HDF5 must be pre-generated**: run the preprocessing pipeline (`src/preprocess/`) first. The file is not in version control.
- **`FreeViewBatch` is legacy**: the active dataset class is `FreeViewInMemory`. `FreeViewBatch` (batch-level HDF5 reads) was an earlier experiment and is not wired into `PipelineBuilder`.
- **`torch.compile` and `InferenceRecorder` are mutually exclusive**: `compilate: True` is skipped if the recorder is enabled.
- **`auto_best_denoise`**: setting `model.pretrained_encoder_path: "auto_best_denoise"` scans all `outputs/` subdirectories for runs with `Phases: ["Denoise"]` and picks the one with the lowest `denoise_error_val`. Requires `outputs/` to exist.
- **Input encoder choice determines many things**: `shared_gaussian` (current default) uses separate Fourier encoders for position, time, and duration with shared random basis matrix `B`. Older variants (`linear`, `fourier`, `nerf_fourier`, `fourier_concat`) are retained in the codebase but are not the target architecture.
- **`SaveCleanX` must precede noise transforms** in the transform list — `PipelineBuilder` sets a flag (`has_save_clean_x`) to automatically also normalise `clean_x` when coord normalisation is applied.
- **PAD_TOKEN_ID = 0.5**: padding value used for masked positions in variable-length sequences. Chosen to sit at the centre of the normalised `[0,1]` coordinate space.
- **Hydra output directories are the reproducibility record**: never edit `outputs/<run>/.hydra/config.yaml` after a run; always re-run with the same config file instead.
